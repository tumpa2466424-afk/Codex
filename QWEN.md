# Locus Coffee Shop — Интернет-магазин кофе

## Обзор проекта
Веб-приложение интернет-магазина кофе с интеграцией Yandex Cloud и YDB. Включает систему авторизации, корзину, оплату (Robokassa), карты (Leaflet), доставку (CDEK), промо-акции и бонусную программу.

## Tech Stack
### Фронтенд
- **Vanilla JS** (ES Modules)
- **Leaflet** — карты (закомментировано, подготовка)
- **CDEK Widget 3.0** — виджет доставки (закомментировано, подготовка)
- **pdfMake** — генерация PDF прайс-листов
- **html2canvas** — генерация наклеек/стикеров
- **CryptoJS** — шифрование для Robokassa

### Бэкенд (Yandex Cloud Functions)
- **Node.js** — серверная логика
- **YDB SDK** (`ydb-sdk`) — серверная БД
- **bcryptjs** — хеширование паролей
- **jsonwebtoken** — JWT авторизация
- **uuid** — генерация уникальных идентификаторов
- **nodemailer** — отправка email
- **Google Calendar API** — синхронизация заказов

### Инфраструктура
- **Yandex Cloud Functions** — бэкенд-функции
- **YDB** — серверная БД (Yandex Database)

## Rules
- Работать только над тем кодом и функционалом, о котором просит пользователь; не затрагивать без разрешения другой код и функционал
- Не удалять файлы без подтверждения
- Не менять API ключи
- Не коммить секреты (JWT_SECRET, API keys, `.env`)
- Перед изменением бэкенда — уточнять у пользователя
- Сохранять LF line endings (UNIX формат)
- Не удалять закомментированные библиотеки (Leaflet, CDEK) — готовятся к подключению
- При редактировании CSS — сохранять CSS переменные и брендовые цвета
- JS: использовать ES Modules синтаксис (`import/export`)
- Не менять стили без запроса
- Сначала объяснять план изменений, потом реализовывать
- Не трогать бэкенд-функции без явного запроса
- Комментарии — только для сложной логики
- Если с первого раза не получается выполнить задачу о которой просит пользователь (сделал, но не получил результата) — необходимо проанализировать зависимости (другие файлы проекта) чтобы понять, нет ли там взаимосвязей влияющих на выполнение задачи и получение нужного результата

## Conventions
- ES Modules (`import/export`)
- CSS переменные для темизации
- Модульная архитектура (разделение фронтенд/бэкенд)
- `.gitattributes`: `text=auto` + LF нормализация

## Заметки
- Некоторые библиотеки (Leaflet, CDEK) закомментированы — подготовка к подключению
- Бэкенд использует JWT с секретом `locus-coffee-super-secret-key-2026`
